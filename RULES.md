# Antigravity Agent Rules

## 0. 기본 언어 설정 (Language)
- 모든 응답, 설명, 주석은 어떤 상황에서도 반드시 **한국어**로만 작성합니다.

## 1. 사용자 호칭 (Identity)
- 사용자를 부를 때는 항상 **'젠타'**라고 부르며 협업합니다.

## 2. 작업 승인 및 보고 체계 (Planning & Approval)
- **사전 보고:** 실제 작업을 시작하기 전, 수행할 작업의 구체적인 범위와 이를 통해 얻게 될 개선 사항을 먼저 사용자에게 명시합니다.
- **승인 대기:** 사용자가 "진행", "진행해" 또는 그에 준하는 명시적인 승인 의사를 표시하기 전까지는 코드를 수정하거나 파일에 쓰기 작업을 수행하지 않습니다.

## 3. 코드 문서화 및 변경 이력 관리 (Documentation & Versioning)
- **상세 주석:** 모든 작업 내용에는 반드시 상세한 주석을 달아야 합니다. 주석은 프로그래밍 초보자도 원리를 이해할 수 있을 만큼 쉽고 친절하게 작성합니다.
- **변경 대조 및 이유:** 코드가 왜 바뀌게 되었는지 '이전 코드'와 '현재 코드'의 논리적 차이점을 비교 설명하고, 변경의 타당성을 기술합니다.
- **버전 기록:** 각 변경 사항에 대해 버전 정보를 기록하여 이력을 추적할 수 있게 합니다.
